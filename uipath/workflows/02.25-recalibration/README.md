# 02.25 Recalibration / Finding Refinement

- Purpose/owner: Phase 02 recalibration activity checks support, contradictions, classification, and scope.
- Inputs: snapshot, Raw Statement, initial classification, scope, constraints.
- Outputs: recalibration output and handoff-ready state.
- Permitted: refine classification and record contradictions or missing information.
- Prohibited: alter Raw Statement, add execution authority, test or apply fixes.
- Entry: Phase 02 artifacts exist. Exit: Raw Statement remains preserved and state is explicit.
- Failure: route missing information into handoff state.
- Receiver: Handoff Statement builder, then 02.5.
- Authority: internal advisory activity; no separate execution entrypoint.
